## О проекте

Данный репозиторий создан в рамках учебной практики студентов 2 курса по направлению «UI-тестирование». В качестве объекта тестирования выбрано веб-приложение **AliExpress** (`https://aliexpress.ru/`).

Проект предполагает выполнение полного цикла тестирования, максимально приближенного к реальным условиям работы в IT-компании, с использованием паттерна **Page Object Model (POM)**.

## Задачи проекта

- Тестирование пользовательского интерфейса (UI-автотесты);
- Проверка ключевых сценариев работы пользователя с сайтом (поиск, фильтрация, корзина, карточка товара);
- Написание тестовой документации (чек-лист, отчёт по практике);
- Организация командной работы с использованием системы контроля версий Git.

## Используемые инструменты

| Инструмент | Назначение |
|------------|------------|
| **Java 17** | Язык программирования |
| **Maven** | Система сборки и управления зависимостями |
| **JUnit 5** | Фреймворк для написания и запуска тестов |
| **Selenide** | Фреймворк для UI-автоматизации (управление браузером) |
| **AssertJ** | Библиотека для читаемых проверок (assertions) |
| **Git + GitHub** | Система контроля версий и хостинг репозитория |
| **IntelliJ IDEA** | Среда разработки (IDE) |

## Этапы выполнения

1. **Проектирование тестов** — составление чек-листа на основе требований;
2. **Согласование с заказчиком (преподавателем)** — утверждение чек-листа;
3. **Написание автотестов** — реализация тестов на Java + Selenide;
4. **Написание документации** — оформление отчёта по практике;
5. **Сдача итогового результата** — защита проекта.
